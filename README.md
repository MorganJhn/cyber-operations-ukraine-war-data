# Replication Data: The Illusion of Completeness

> **How Single-Database Studies of Cyber Conflict Produce Systematically Incomplete Pictures**

**Author:** Morgan Johanson  
**Affiliation:** The Morgan Johanson Foundation  
**Status:** Working paper — under embargo. Not for citation or redistribution without author permission.

---

## Overview

This repository contains the replication data and analysis code for a study examining how reliance on single databases (CISSM, CSIS, CFR) produces systematically incomplete pictures of cyber operations in the Russia-Ukraine war (2022–2026).

The study analyzes **442 cyber incidents** (February 2022–December 2024) coded across 59 variables, demonstrating that:
- No single database captures more than a fraction of documented incidents
- Cross-source overlap is near-zero, with each database contributing unique cases
- Attribution patterns (state vs non-state) vary dramatically across sources
- Temporal and event-type distributions are database-dependent, not conflict-inherent

## Replication Instructions

### Prerequisites

- Python 3.10+
- Required packages:

```bash
pip install -r requirements.txt
```

### Running the Analysis

From the repository root:

```bash
# Run all analysis scripts in sequence
for script in analysis/build_*.py; do
    echo "Running $script..."
    python3 "$script"
done

# Validate results
python3 analysis/validate_attribution_coding.py
```

### Expected Output

After running all scripts, the `output/` directory will contain:
- 6 JSON files in `output/stats/`
- 3 LaTeX table fragments in `output/tables/`
- 7 figure files in `output/figures/`
- 2 apparatus files in `output/apparatus/`

These should match the pre-committed outputs included in this repository.

## Data Description

### paper_analysis_corpus.csv

The analysis corpus contains 442 cyber incidents with complete date records from February 2022 to December 2024, sourced from three databases: CISSM (Georgetown CSET), CSIS (Center for Strategic and International Studies), and CFR (Council on Foreign Relations). 58 variables are derived from the raw source databases plus enrichment and normalization, including:

| Variable | Description |
|----------|-------------|
| `source` | Source database (`cissm`, `csis`, `cfr`) |
| `date` | Incident date (YYYY-MM-DD) |
| `actor` | Perpetrator name |
| `actor_country` | Perpetrator state affiliation |
| `target_country` | Target state |
| `event_type` | Primary event category |
| `event_subtype` | Secondary event category |
| `motive` | Attributed motive |
| `attribution_method` | How attribution was determined |
| `actor_type` | Actor classification (state_russian, hacktivist_prorussian, etc.) |
| `include_in_analysis` | Whether incident is in the analysis set |

### Intercoder Reliability

The `intercoder_summary.csv` file reports consistency rates for attribution coding across seven methods, based on double-coding of a random sample by a second coder.
